Häufige Ursachen nach Projekt-Duplikation
Wenn du den gesamten Ordner kopiert hast, wurden wahrscheinlich alte Build-Artefakte übernommen.
✅ Lösung:
rm -rf .next
rm -rf node_modules
rm bun.lockb
bun install
bun dev

---------------------
2. .env.local enthält falsche URLs / Keys
Du hast: Environments: .env.local
Wenn dort z. B.:

alte Datenbank-URLs
falsche API-Endpunkte
Vercel-URLs
localhost Ports
Supabase/Firebase Keys

stehen, kann die App beim Laden hängen.

3. API wartet endlos

Sehr häufig:
Eine Server Component oder API Route macht:
await fetch(...)
oder
await db.query(...)
und die Anfrage hängt.
Teste:

Browser DevTools → Network Tab
Terminal prüfen ob Requests hängen
console.log() in page.tsx oder API Routes setzen
---------------
4. 🔴 Port-Konflikt oder alter Dev-Server
Manchmal läuft noch ein anderer Prozess auf 3000.
Teste:
✅ lsof -i :3000
oder einfach:
✅ bun dev --port 3001

5. Turbopack / Next.js hängt
Falls du Next.js benutzt:
Teste mal ohne Turbopack:
✅ next dev
oder:
✅ bunx next dev

6. Symlink-/Pfadproblem durch Kopieren

Wenn der neue Ordner in einem synchronisierten Ordner liegt (Dropbox, iCloud, OneDrive), wird Next/Bun manchmal extrem langsam.

Besonders problematisch:
Desktop
Documents
iCloud Drive
Netzwerkordner

Teste das Projekt mal z. B. in:
C:\dev\meinprojekt
oder
~/Projects/
Schnellster Debug-Test

✅ Lösung in 3 Schritten:
Schritt 1: Google Cloud Console prüfen
Gehe zu: https://console.cloud.google.com/apis/credentials

Prüfe ob die Credentials existieren:

Suchst du nach GOOGLE_CLIENT_ID und GOOGLE_CLIENT_SECRET

Prüfe die Authorized Redirect URIs:

Müssen exakt sein:

text
http://localhost:3000/api/auth/callback/google
http://localhost:3000/api/auth/callback/google? (mit Fragezeichen)
Prüfe ob OAuth Consent Screen konfiguriert ist:

User Type: External (für Entwicklung)

Test users hinzugefügt (deine Email)

Schritt 2: Korrigiere deine .env.local
env
# Deine aktuelle .env.local
GOOGLE_CLIENT_ID=deine_google_client_id.apps.googleusercontent.com  # ← Muss komplett sein
GOOGLE_CLIENT_SECRET=dein_google_client_secret  # ← Keine Leerzeichen, keine Anführungszeichen
Entferne die .apps.googleusercontent.com Endung NICHT - sie gehört zur Client ID!

Schritt 3: Für GitHub auch prüfen
Falls GitHub auch nicht funktioniert:

Gehe zu: https://github.com/settings/developers

OAuth Apps → deine App

Homepage URL: http://localhost:3000

Authorization callback URL: http://localhost:3000/api/auth/callback/github
